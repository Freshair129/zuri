## Objective
Implement the full Kitchen Ops flow for restaurant/culinary school use case:
- [x] Fix schema bug (`Ingredient` + `IngredientLot` missing `tenantId`)
- [x] Product Management back-office UI (add/edit menu products + Recipe linking)
- [x] Recipe Builder UI (create/edit recipes with ingredient picker)
- [x] Ingredient Management UI (CRUD + goods receipt / lot creation)
- [x] POS → Stock Deduction (auto-deduct FEFO ingredients when order confirmed)
- [x] Kitchen Dashboard wiring (fix stale API paths)
